# 环形缓冲区/队列模式

​	**环形缓冲区（Ring Buffer）**，又称**循环队列（Circular Queue）**，是一种高效的数据结构，特别适用于**生产者-消费者模型**、**实时流数据处理**以及**嵌入式系统**中。它的核心思想是：**将一块固定大小的连续内存空间逻辑上首尾相连，形成一个“环”**。通过两个指针（读指针和写指针）在环上循环移动来管理数据的写入和读取，从而避免了传统线性队列在数据出队后需要移动大量元素或频繁分配/释放内存的问题。



## 一、核心原理与结构

### 1、基本组成

* **缓冲区**：预分配的固定大小连续内存块。为了优化性能，大小通常设为 2n2n （2的幂），以便使用位运算代替取模运算。
* **头指针（写指针write_index）**：指向下一个可写入空间的指针。
* **尾指针（读指针read_index）**：指向下一个可读取数据位置的指针。
* **容量（Capacity）**：环形缓冲区的容量。



### 2、工作流程

* **写入**
  - 数据存入写指针（write_index）指向可写入的位置。
  - 写指针（write_index）向前移动：**write_index = (write_index + 1) % capacity。**
  - 当写指针追上读指针时缓冲区满，可以选择覆盖数据存储或者丢弃新数据。

* **读取**
  - 从读指针（read_index）指向的位置读取数据。
  - 读指针（read_index）向前移动：**read_index = (read_index + 1) % capacity。**
  - 当读指针（read_index)追上写指针（write_index）此时缓冲区为空。
* **缓冲区空与满的区分方法**
  * **牺牲一个单位容量存储空间**
    * 当读指针（read_index）等于写指针（write_index）时即可能是缓冲区满也可能是缓冲区空，这样是不能区分是满还是空的，所以可以牺牲一个单位的容量空间，容量（capacity）变为**capacity - 1**。
    * 当**（write_index + 1） % capacity == read_index**时缓冲区为满。
    * 当**write_index = read_index**时缓冲区为空。

  * **增加缓冲区存储计数器（count）**
    * 当**count == 0**时缓冲区为空。
    * 当**count == capacity**时缓冲区为满。

  * **增加读写标志位r_or_w_flag**
    * 当**read_index == write_index**时。
    * 如果最后一次操作是写操作时，缓冲区为满。
    * 如果最后一次操作是读取操作时，缓冲区为空。
