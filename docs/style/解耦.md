# 解耦

## 一、高层 - > 底层注入

**依赖注入就是“对象不负责创建自己的依赖，而是由外部环境（容器）在运行时将依赖传递给它”。**

**传统模式：** 上层模块主动去调用/创建下层模块。

**依赖注入模式：** 上层模块被动接收下层模块。`App` 不需要知道 `BSP` 的具体实现类名，它只需要知道“我会有一个驱动对象传进来”。

其中需要关注的角色：

1、**注入者：**

- 选择在**`main.c`**。它负责实例化所有对象（创建 `bsp` 实例，创建 `app` 实例），并将它们组装在一起。

2、**被注入者：**

- **`App`**：它只负责定义“我需要什么样的接口”，然后坐等 `main.c` 把接口传给它。



## 二、底层- > 高层回调

这是一种“反向依赖”的解耦方式，常用于事件处理。

 下层不直接调用上层的函数，因为下层不应该依赖上层。下层定义一个“注册接口”，上层将自己的处理函数注册给下层。当硬件事件发生时，下层调用这个函数指针。

例如下层传感器获取数据，它只需要获取数据，内部发生事件时产生回调通知上层数据处理者对获取到的数据进行处理。
