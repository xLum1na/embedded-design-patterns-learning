# 知识点

## 在这个示例中，有以下几点知识点：

### 1、状态机的使用

​	创建了2个状态机，一个按键状态机负责进行按键事件发送，一个led状态机负责通过按键发送的事件进行对应状态的变化。

​	在按键状态机的实现方法上使用了switch + 回调的方式进行实现。在led状态机的实现方法上使用了查表法（函数指针法）的方式进行实现。

### 2、事件驱动框架

​	通过按键回调函数进行对应按键事件的发送，按键产生的事件被放入一个事件队列中，led状态机通过对这个事件队列进行监听，从而进行对应状态的切换。

### 3、串联状态机的实现（按键与led状态机）

​	串联式状态机的实现有以下几个要求：(1)状态机间是共同运行的。(2)状态机间有数据的传输。

### 4、生产者消费者模式（按键生产事件，led消费事件）

​	按键负责生产事件，led负责消费按键产生的事件。

### 5、观察者模式

​	此模式着重点在于消息是同步发送与接收的，因为被观察者与观察者间要直接进行耦合，被观察者产生消息后观察者要直接进行消息的提取，被观察者产生消息后通过回调函数进行消息传输。

## 后续可学习知识点有：

### 1、使用事件总线与发布-订阅模式进行按键与led两个状态机间的解耦

​	使用事件总线的形式类似于发布/订阅模式，都在2个文件间建立了一个共同需要的文件，按键发送事件到总线上等同于按键发布消息到共同包含的事件分发服务中。led接收事件总线中的信息等同于led订阅事件分发服务。

### 2、使用分层状态机（HSM）进行状态机开发

​	led的状态作为状态机系统的超状态，按键状态机作为子状态。
