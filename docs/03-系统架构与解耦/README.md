### **3. 系统架构与解耦（7个模式）**

| 模式名称         | 学习优先级   | 核心应用场景                                                 | 学习状态 | 掌握程度 | 最后复习 | 备注 |
| ---------------- | ------------ | ------------------------------------------------------------ | -------- | -------- | -------- | ---- |
| **桥接模式**     | 🟡 第三优先级 | 分离抽象与实现，使两者独立变化（协议与介质解耦、UI与显示驱动） | ☐ 未学   | ★☆☆☆☆    | —        |      |
| **适配器模式**   | 🟡 第三优先级 | 解决已有组件接口不兼容问题（驱动接口统一、旧代码复用）       | ☐ 未学   | ★☆☆☆☆    | —        |      |
| **模板方法模式** | 🟡 第三优先级 | 定义算法骨架，子类实现具体步骤（驱动初始化框架、设备操作流程） | ☐ 未学   | ★☆☆☆☆    | —        |      |
| **装饰器模式**   | 🟢 第四优先级 | 动态添加职责（为数据流加CRC/加密/日志）                      | ☐ 未学   | ★☆☆☆☆    | —        |      |
| **外观模式**     | 🟡 第三优先级 | 为复杂子系统提供统一高层接口（封装算法库、协议栈）           | ☐ 未学   | ★☆☆☆☆    | —        |      |
| **组合模式**     | 🟢 第四优先级 | 树形结构表示部分-整体层次（设备树、GUI控件树）               | ☐ 未学   | ★☆☆☆☆    | —        |      |
| **享元模式**     | 🟢 第四优先级 | 共享大量细粒度对象（字体库、常量表、正弦表）                 | ☐ 未学   | ★☆☆☆☆    | —        |      |
