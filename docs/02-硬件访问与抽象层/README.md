### **2. 硬件访问与抽象层（6个模式）**

| 模式名称           | 学习优先级   | 核心应用场景                                                 | 学习状态 | 掌握程度 | 最后复习 | 备注           |
| ------------------ | ------------ | ------------------------------------------------------------ | -------- | -------- | -------- | -------------- |
| **硬件代理模式**   | 🔴 第一优先级 | 为硬件创建软件代理，封装底层操作细节（传感器驱动、外设驱动） | ✅ 已学   | ★★★★☆    | —        |                |
| **硬件适配器模式** | 🟡 第三优先级 | 转换硬件接口，使不兼容的双方能协同工作（旧代码复用、接口转换） | ☐ 未学   | ★☆☆☆☆    | —        |                |
| **中介者模式**     | 🟡 第三优先级 | 通过中介对象封装模块间交互，降低耦合（网关设备、多模块协调） | ☐ 未学   | ★☆☆☆☆    | —        |                |
| **去抖动模式**     | 🔴 第一优先级 | 用软件消除机械开关抖动（按键、限位开关）                     | ☐ 未学   | ★☆☆☆☆    | —        | 常与状态机结合 |
| **轮询模式**       | 🔴 第一优先级 | 循环检查设备状态（极简系统、非实时任务）                     | ☐ 未学   | ★☆☆☆☆    | —        |                |
| **Bit Bang模式**   | 🟢 第四优先级 | 用软件模拟硬件通信协议（I2C/SPI模拟、资源受限场景）          | ☐ 未学   | ★☆☆☆☆    | —        |                |
